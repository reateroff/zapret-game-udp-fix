# zapret-game-udp-fix

Фикс запрета для FACEIT и CS2 на EU серверах (OVH, Hetzner и др.)

Основано на [zapret-discord-youtube](https://github.com/Flowseal/zapret-discord-youtube) v1.9.7b

---

## Что поменяно

В оригинале game filter ссылался на пустой `ipset-all.txt` - по факту не работал, ни один игровой пакет не обрабатывался.

Сейчас UDP правило для игр:
```
--dpi-desync=fake
--dpi-desync-repeats=10
--dpi-desync-cutoff=n4
--dpi-desync-ttl=5
--dpi-desync-fake-unknown-udp=stun.bin
```

---

## Как это работает

При подключении к серверу ТСПУ анализирует первые UDP пакеты и определяет, что это игровой трафик. После этого блокирует соединение - handshake проходит, но дальше сервер перестаёт отвечать.

Запрет отправляет fake-пакет перед каждым настоящим. ТСПУ видит fake первым, принимает его за реальный трафик и пропускает настоящий пакет. Сервер при этом fake не получает - пакет умирает на полпути из-за низкого TTL.

`stun.bin` вместо `quic_initial` - 100 байт вместо 1200. ТСПУ палит fake по размеру, а STUN пакет по размеру не отличается от игрового.

`--dpi-desync-cutoff=n4` - десинк работает только первые 4 пакета соединения, дальше трафик идёт напрямую. На пинг в игре не влияет.

---

## Как запустить

1. `general (SIMPLE FAKE).bat` от имени администратора
2. Game Filter в `service.bat` → UDP

---

## TTL

`--dpi-desync-ttl=5` - подобрано под провайдера Interra. У другого провайдера ТСПУ стоит на другом хопе, поэтому значение будет отличаться.

Чтобы узнать свой TTL:
```
tracert <IP сервера>
```

Хоп с `* * *` - это ТСПУ, его номер = нужный TTL.
